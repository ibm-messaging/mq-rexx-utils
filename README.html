MQ Configuration difference tool MQConfigDiff

This tool provides rexx execs to take output from CSQUTIL or runmqsc and create HTML pages so you can compare and contrast the differences on MQ object and their attributes.

You will need rexx installed on your machine. These rexx programs have been tested in Redhat(R) V7 and Windows(R) V10.

Differences are highlighted. You can display all attributes, or just different attributes. The example below is a cut down version showing just the queue manager and a few local queues.

Data processed on 16 Apr 2016 10:31:16

Summary of objectsall attributesdifferences
QMGR All details Differences
QLOCAL All details Differences

You can use this to compare queue managers and see which attributes are different; or compare attributes from one queue manager at different points in time to see what has changed between the two times.

If you click on the links, the columns have names

  • MQP1.out.)MQP1, This mean the queue manager )MQP1 came from file MQP1.out
  • n7.defs.txt.out.colin, this means queue manager colin from the file 7.defs.txt.out.
This allows you to pass in files like MQP1.March20.out and MQP1.March30.out and see the differences.

Syntax

CSQUTIL

rexx mqzos.rexx infile < outfile >

where infile might be MQP1.defs and outfile will default to infile.out

runmqsc

rexx mqdist.rexx infile < outfile >

where infile might be default.defs and outfile will default to infile.out

Compare the files

The above processing converts the input files to one keyword value pair per line. to create the html

rexx to html.rexx output1 output2 ... where the output1 and output2 etc are the output files from the mqzos and mqdist commands.

Then browse the aStart.html file in your directory.

The execs will erase any files before recreating them, so rename, or take a copy of the files you want to keep.
